# Overview

For any given notification, we recommend you view it at [https://my.extole.com/notifications](https://my.extole.com/notifications). There, you can view more details about the notification and optionally snooze it for up to 30 days in the future.

| We couldn’t understand **event name** event from **app name**. Please check the **event name** event you’re sending has the required data attributes. | Extole will sometimes manipulate the event you're sending before processing it. You will receive this notification if there's something broken in the manipulation/translation step between reading the event you're sending and having our system actually process it. | Check to make sure that nothing about the events you're sending Extole has changed (e.g. the events aren't missing any data attributes). | If everything looks okay from your side, Extole may need to check:Are we assuming data attributes are being sent on events that aren't actually being sent?Are we attempting to do an invalid operation? |

You will receive this notification if we're unable to assign a value to a specific event. | Check to make sure that nothing about the events you're sending Extole has changed (e.g. the events aren't missing any "value" data attributes such as cart value or funding amount). | If everything looks okay from your side, Extole may need to check:Are we assuming "value" data attributes are being sent on events that aren't actually being sent? |

You will receive this notification if we're unable to understand one of the data attributes being sent on one of your events. | Check to make sure that nothing about the events you're sending Extole has changed (e.g. the events aren't missing any data attributes). | If everything looks okay from your side, Extole may need to check:Did we write custom code to extract a data attribute from an event, and are there any bugs in that custom code? |
| We couldn’t execute the **action name** action for the event name event in campaign **campaign name**. | We couldn’t execute the expression action for the **event name** event in campaign **campaign name**. Please check the **event name** event you’re sending has the required data attributes. | **Extole can set up custom actions to be taken off of an event. | We couldn’t execute the **condition name** condition for the **event name** event in campaign **campaign name**. You will receive this notification if we were unable to run a reward rule on a specific event. | Check to make sure that nothing about the events you're sending Extole has changed (e.g. the events aren't missing any data attributes).Check to make sure the person isn't blocked, and nor are the actions leading to the event.Understand if a make-good needs to be created in order to issue the person their reward. | If everything looks okay from your side, Extole may need to check:Did we create a custom reward rule, and has the custom code changed at all?Is the person or are the actions leading to the event blocked?Do we need to do a make-good in order to issue the person their reward? |

You will receive this notification if we were unable to issue a reward for a specific event. | Check to make sure that there is a reward supplier associated with all of your rewardable events.Understand if a make-good needs to be created in order to issue the person their reward. | If everything looks okay from your side, Extole may need to check:Do we need to do a make-good in order to issue the person their reward? |
