# Overview

The Reward Transitions report provides a detailed list of rewards that have undergone state transitions during a specified time period. A state transition refers to a change in the status of a reward, such as from earned to fulfilled or from fulfilled to failed. By analyzing this report, you can track the movement of rewards between different states, identify patterns, and ensure efficient reward management.

# How This Report Can Help You

- Monitor the progress of rewards from the moment they are earned to their final fulfillment or status change.
- Identify any delays or issues in the reward fulfillment process.
- Gain insights into the efficiency of your reward suppliers and types.
- Understand participant behaviors and engagement levels based on reward state transitions.

# How to Run the Report

1. Select the desired time range for the report, or specify a custom timeline for a more tailored analysis.

2. Choose the program label or campaign ID to focus on specific programs or campaigns, or leave it blank to analyze all programs.

3. Refine the results by selecting specific reward supplier types or reward states, such as earned, fulfilled, or failed.

# Report Results

The Reward Transitions report offers a comprehensive overview of rewards that have undergone state transitions within the specified time period. The following fields are included in the report:

- Reward ID: A unique identifier for each reward.
- Reward State Transition Time: The date and time when the reward transition occurred.
- Partner Reward ID: Partner-specific identifier for the reward (if applicable).
- Face Value Type: The type of currency or value associated with the reward (e.g., USD).
- Face Value: The amount or value of the reward.
- Campaign ID: Unique identifier of the campaign associated with the reward.
- Campaign Name: The name of the campaign.
- Reward Supplier Type: The type of reward supplier, such as CUSTOM_REWARD or TANGO_V2.
- Reward Supplier ID: Unique identifier for the reward supplier.
- Partner Reward Supplier ID: Partner-specific identifier for the reward supplier (if applicable).
- Passed Through Reward State: The previous state of the reward before the transition.
- Current Reward State: The current state of the reward after the transition.
- Reward Name: The name or description of the reward.
- Person ID: Unique identifier of the participant associated with the reward.
- Email: Email address of the participant.
- Partner User ID: Partner-specific identifier for the participant (if applicable).
- First Name: First name of the participant.
- Last Name: Last name of the participant.

By analyzing the data presented in the Reward Transitions report, you can efficiently manage rewards, address any potential issues, and ensure a seamless experience for your participants throughout the reward fulfillment process. This report empowers you to make data-driven decisions and optimize your reward programs for maximum participant satisfaction.
