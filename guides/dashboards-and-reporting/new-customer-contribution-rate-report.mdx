# Overview

The **New Customer Contribution Rate** report provides an estimated ratio of new customers driven by Extole programs compared to your total new customer acquisition. It helps you understand the scale of acquisition through Extole relative to other channels.

# How to Run the Report

1. **Select a time range**: Choose from predefined options such as Last Day, Last Week, or Last Month, or specify a custom range.
2. **Choose the program**: Select a specific program or choose all programs to include all programs in the report.

This report defaults to the necessary fields. No additional customization is typically needed.

# Report Results

When you run the report, you’ll see a table with the following metrics:

- **Period End**: The last date of the selected time period.
- **Attributed New Customers**: Number of new customers attributed to an Extole program.
- **All New Customers**: Total number of new customers across all marketing channels.
- **Customer Contribution %**: Attributed New Customers ÷ All New Customers.
- **Attributed New Transactions**: Number of transaction events attributed to an Extole program.
- **All New Transactions**: Total number of transaction events across all marketing channels.
- **Transacted Contribution %**: Attributed New Transactions ÷ All New Transactions.

This data helps you understand what percentage of your new customers are being driven by Extole programs.

# How This Report Is Calculated

Most clients send all relevant events (e.g., conversions, account openings) to Extole. This allows us to compare new customers attributed to Extole with your total acquisition. These events are configured by your team during implementation.

## Definitions

- **Customer**: The event at which a user becomes a customer. This may differ from the reward-triggering event.
- **Transacted**: The event at which a user completes a qualifying payment. This may also differ from the reward-triggering event.

If the data returns as 100% or looks incorrect, contact your CSM to review how the **Customer** and **Transacted** aliases are implemented and whether you are sending all events to Extole.
