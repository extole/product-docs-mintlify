# Overview

In the course of supporting customers and running your referral campaign, you and your support team may need to manually create referrals or other events.

## Why would I need to manually create an event?

A primary use case for manual event creation is when a customer reaches out to you about a missing reward. You may be unable to find the referral, sign-up, etc. that would have triggered the reward in your My Extole account, however, you can see in your own system that the event occurred. In this case, you can manually create the event in My Extole.

# How to Create an Event

1. **Go to User Support**. In the left-hand side bar of your My Extole account, open the User Support dropdown and click on [Create Event](https://my.extole.com/create-event).
2. **Fill Out the Create Event form**. You will see a number of fields to fill out, all of which are pictured and described below.

| **Field** | **Description** |

| Program (required) | The program in which the event occurred. |
| Campaign (required) | The campaign in which the event occurred. |
| Event (required) | The type of event you are creating. This will likely be an "outcome" event -- a registration, conversion, account opening, subscription purchase, quote submission, etc. If the event you're looking for does not show up, that means it is not an enabled event for the campaign in which you're creating an event. Please contact your CSM or Extole's Support team if you need help creating an event. |
| Email (required) | The email address of the friend who made the registration/conversion |
| Promotion Source | The promotion that the advocate viewed and then shared from -- this is the promotion source that you want this event to track to. |
| Partner Conversion ID | The unique identifier for the event, such as an order number or account number. This field has a max length of 38 characters. |
| Event Time | The date and time when the event occurred. |
| Event Value | The value of the event, such as a cart value or funding amount. |
| User: Email | The email address of the person you're creating the event on behalf of. | User: Partner User ID | The unique identifier (e.g. account ID, member ID) for the person you're creating the event on behalf of. | Advocate: Email | The email address of the advocate if this is a referred event (such as a registration, conversion, account opening, etc.). |
| Advocate: Partner User ID | The unique identifier (e.g. account ID, member ID) for the advocate if this is a referred event (such as a registration, conversion, account opening, etc.). |
| Advocate: First Name | The first name of the advocate if this is a referred event (such as a registration, conversion, account opening, etc.). |
| Advocate: Last Name | The last name of the advocate if this is a referred event (such as a registration, conversion, account opening, etc.). |
| Additional Event Data | A list of optional parameters to include with the referral. The most common parameter is coupon_code, used to indicate the coupon a friend used on their purchase event. |

1. **Approve the Event**. If the event you've created has a pending period (most common for conversion events), the event will be placed in a pending state. If a reward is issued on the event and you are ready to send the reward, you need to approve the event. To do this, in the sidebar, expand the User Support category and click [Approvals](https://my.extole.com/events/approvals).   
​  
You can approve events one at a time by clicking Approve to the right of an event in the table, or you can approve multiple events at once by clicking the checkbox next to the events you want to approve and clicking Approve Selected at the top of the table.
