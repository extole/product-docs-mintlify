# Overview

We offer the ability for your consumers to add a reward as a pass in their mobile wallets! Advocates and Friends can do this from their earned reward emails. Simply toggle on "Mobile Wallet Button" and set your "Mobile Wallet Button Text" in the Overview tab.

![Screen_Shot_2021-04-06_at_12.01.14_PM.png](https://extole-5ef307a0e5b1.intercom-attachments-7.com/i/o/syy27wia/1421734676/e345360b87ef4f45ef7e79d99e6d/1500008085622?expires=1778306400&signature=dd879c12a81668f5970f19501b0d47259ad5b5e84fac6544cb10a29026d686fe&req=dSQlF859mYdYX%2FMW3nq%2BgUmpP3WuLFidOtnCVPeXezz%2BWYJpzdi3zAcjyDKC%0AjLpvmzsIBSdviwNgWT8w2J7RMFY%3D%0A)

This feature has 3 distinct pieces:

- An iOS native mobile wallet pass, compatible with iOS devices
- An iOS-issued mobile wallet pass, compatible with Android devices using the WalletPasses app
- A desktop landing page prompting users to open their pass on their mobile device by scanning a QR code

This article will walk through the creative elements required to make use of the wallet passes feature.

| **Creative Variable Name** | **Uses** |
| --- | --- |
| Company Logo Image | Shows in the upper left-hand corner of the microsite (see below) to reinforce your brand. Also typically used as the Pass Logo. |
| WalletPasses Button | Prompts users to download their pass to their mobile Android device using the WalletPasses app. This is visible on the mobile reward pass destination. |
| Pass Logo | Typically the same as the Company Logo; shows in the upper left-hand corner of the pass. |
| Pass Strip | Can call out what the pass can be used for. In our example, it's the image that says "Take 20% off your next purchase!". |
| Label for Name on Pass | Typically left as "Name"; signifies who the pass belongs to. |
| Label for Coupon Code on Pass | Typically left as "Coupon Code"; calls out the coupon code associated with the reward. |
| Label for Terms on Pass | On the back of the pass, how the Terms and Conditions should be referred to. Typically left as "Terms and Conditions". |
| Label for Updates on Pass | On the back of the pass, how the Updates section should be referred to. Typically left as "Updates". |
| Text for Updates on Pass | Any updates you'd like to call out on the back of the pass. |
| Pass Barcode Type | Valid Values: QR, PDF_417, AZTEC, CODE_128 |
| Text for 'Back' Button on Back of Pass | On the back of the pass, what the user clicks on to return to the front of the pass. Typically left as "Back". |
| Wallet Pass Font Family | What font family the pass fonts belong to. |

# Reward.pkpass

| **Creative Variable Name** | **Uses** |
| --- | --- |
| Reward Pass Description | A description for the pass; usually left as the Company Name. |
| Reward Pass Organization Name | The name of the organization issuing the pass (that's you!). |
| Icon Image URL | The URL where the icon image is hosted. This defaults to your company icon, as set in the Overview tab. |
| Logo Image URL | The URL where the logo image is hosted. This defaults to your company logo, as set in the Overview tab. |
| Strip Image URL | The URL where the strip image is hosted. This defaults to the Pass Strip, as set in the Overview tab. |
| Background Color RGB | The background color of the pass. Use Syntax: redValue,greenValue,blueValue e.g. 255,255,255 |
| Foreground Color RGB | The foreground color of the pass. Use Syntax: redValue,greenValue,blueValue e.g. 255,255,255 |
| Label Color RGB | The color for the text labels on the pass. |
| Barcode Type | Valid Values: QR, PDF_417, AZTEC, CODE_128 |
| Label for Coupon Code on Pass | Typically left as "Online Code"; calls out the coupon code associated with the reward. |
| Label for Name on Pass | Typically left as "Name"; signifies who the pass belongs to. |
| Label for Terms on Pass | On the back of the pass, how the Terms and Conditions should be referred to. Typically left as "Terms and Conditions". |
| Terms Text | The terms and conditions of using the pass. |
| Pass Updates Label | On the back of the pass, how the Updates section should be referred to. Typically left as "Updates". |
| Back Field Change Message | This is the text value for the alert that is displayed when the value associated with the Updates field is updated. The change value text must contain the escape value %@, which is replaced with the field’s new value. |
